# 烛微 ZhuWei：产品介绍

![烛微 Logo](../frontend/brand/logo.svg)

烛微（ZhuWei）是一套面向安全运营、应急响应和资产风险跟踪的本地化漏洞情报工作台。它把多源漏洞采集、产品库沉淀、告警筛选、关注资产命中、AI 辅助分析、POC/EXP 产出和日报周报聚合在同一个控制台里，让团队可以更快判断“这条漏洞是否和我有关、风险高不高、下一步该做什么”。

## 产品定位

本产品不是单一 RSS 阅读器，也不是只做 CVE 列表展示的漏洞库。它的核心目标是把分散情报转成可执行的安全工作队列：

- 对外部漏洞源持续采集，统一入库、去重、搜索和追踪。
- 将漏洞对齐到产品库，沉淀“产品-漏洞”关系和本地命中数量。
- 对高价值漏洞进行规则过滤和质量评分，减少噪声。
- 对关注产品命中的 high/critical 漏洞自动触发分析。
- 调用 Claude Code + DeepSeek 生成漏洞研判、根因、攻击面、POC/EXP、参考链接和源码线索。
- 提供分析队列、失败重跑、取消任务、源健康、报表和消息中心，方便持续运营。

## 品牌概念

Logo 由四个元素组成：

- 盾牌：代表防护边界和本地情报中枢。
- 雷达弧线：代表持续监听外部漏洞信号。
- 中央 V 形光束：代表 Vulnerability 和优先级聚焦。
- 红色节点：代表被识别出的高风险利用点。

主色使用深色盾牌、蓝色情报流、青绿色监控信号和琥珀色处置提示，既贴近安全产品的专业感，也避免纯蓝色仪表盘的同质化。

## 核心能力

### 多源情报采集

当前内置数据源包括：

- CISA KEV
- BIU RSS 漏洞情报
- BIU 产品库
- NVD Recent CVE
- 长亭 VulDB
- OSCS 开源安全情报
- 微步漏洞情报
- Seebug 漏洞库
- AVD 高危漏洞
- CNVD 最新列表
- Doonsec 微信 RSS
- 启明星辰安全通告
- Apache Struts2 Security Bulletins

数据源分为 regular 和 slow 两组。regular 默认每 30 分钟执行一次，slow 默认每天 10:00 和 18:00 执行一次。

### 产品库与关注资产

BIU 产品库支持流式采集和页级入库，不需要等待全部页采完再写库。用户可以从产品库或漏洞条目中关注资产；后续新漏洞命中关注产品时，系统会自动记录匹配并触发分析。

当前版本新增产品归属解析能力：

- `products` 表保存产品目录和产品统计。
- `product_vulnerabilities` 表保存产品与漏洞的多对多关联。
- 漏洞入库后优先通过源产品字段、标题、描述和产品库名称做本地直接匹配。
- 新告警无法本地判断产品时，可调用 DeepSeek Flash 做只面向产品归属的联网搜索分析。
- 产品库卡片展示外部产品页、最新 3 条关联漏洞和本地命中数量。

### 告警规则

告警规则支持：

- 最低漏洞等级过滤。
- 白名单关键词。
- 黑名单关键词。
- CVE 去重。
- 告警时间窗口。
- 跳过关键词过滤模式。

漏洞会先入库，再由告警规则决定是否进入待处理队列。

### AI 漏洞分析

分析链路会调用 Claude Code，并通过 DeepSeek Anthropic 兼容接口完成漏洞研判。输出内容包括：

- 漏洞摘要。
- 影响产品。
- 根因分析。
- 攻击面分析。
- POC 可用性与非破坏性验证脚本。
- EXP 利用条件和缓解建议。
- 参考链接和源码仓库线索。
- 置信度与质量评分。

AI 链路分为两类任务：深度漏洞分析默认使用主分析模型；产品归属识别使用轻量 DeepSeek Flash 模型，只搜索对应产品，不生成 POC/EXP。

分析任务支持并发控制、默认优先级、取消运行中任务、失败分类统计和失败任务一键重跑。

### 情报质量评分

质量评分满分 100，综合考虑：

- 严重程度：25 分。
- POC：20 分。
- EXP：10 分。
- 在野利用：15 分。
- 关注资产命中：15 分。
- 数据源数量：5 分。
- 新鲜度：10 分。

接口 `GET /api/alerts/scored` 可返回按综合评分排序的告警，`GET /api/vulnerabilities/{id}/threat-score` 可查看单条评分详情。

### 源健康与运营报表

源健康中心提供每个数据源的最近成功/失败时间、平均耗时、错误类型统计、连续失败次数和入库趋势。

日报/周报接口提供：

- 新增漏洞等级分布。
- POC/EXP 产出。
- 关注产品命中。
- 新增告警。
- 待确认告警。
- 分析完成数。
- 分析失败数。
- 新产品数。

## 适用场景

- 安全运营团队日常漏洞情报值守。
- 应急响应前的漏洞影响面研判。
- 关注产品、供应链组件和重点资产的风险跟踪。
- 高危漏洞 POC/EXP 信息归档。
- 本地化、轻量级、可二次开发的漏洞情报平台。

## 系统边界

- 本产品默认使用本地 SQLite 存储，适合个人、小团队或单机部署。
- AVD、CNVD 等源可能受登录态、WAF、验证码和配额影响，需要维护 Cookie 或会话参数。
- AI 分析依赖 Claude Code 命令行和 DeepSeek API Key；未配置时仍可采集、搜索、告警，但无法完成自动深度分析。
- 当前登录 token 每次服务启动随机生成，并在控制台输出；服务重启后旧浏览器会话会失效。
